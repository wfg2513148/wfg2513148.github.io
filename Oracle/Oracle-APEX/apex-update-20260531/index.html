从 APEXLang 到 AI Agents，APEX 正在成为新一代 AI 开发工作台

发表于 2026-05-31 更新于 2026-06-01 分类于 Oracle ， Oracle APEX 本文字数： 2.8k 阅读时长 ≈ 10 分钟

如果你 5 月只记住一件事，我建议不是“Oracle APEX 26.1 发布了”，而是 APEX 的开发方式开始变了。

以前大家谈 APEX，更多是在说页面怎么搭、表单怎么做、报表怎么配、流程怎么跑。但自从今年以来，社区讨论的重心明显挪了位置：开始谈文件化、版本控制、AI智能体、自然语言报表、DevOps、自动化测试、私有大模型接入、团队协作。说得再直白一点，Oracle APEX 已经越来越不像“拖拖拽拽把页面拼出来的低代码开发工具”，而更像一个 企业级应用敏捷开发底座。而且这个底座，开始自带 AI、自带工程化、自带协作感。这才是2026年5月最值得看的地方。

本次统计口径说明

  • 统计来源：oracleapex.cn
  • 统计范围：2026年5月收集的 APEX 博客译文
  • 最终纳入：72条

这72条里，5月最强的主线其实非常集中：

  • APEX 26.1 发布与工程化变革（16）
  • AI / Agent / LLM 能力（14）
  • 界面、交互与用户体验（14）
  • 报表、搜索与数据交互（8）
  • 安全、认证与会话控制（7）
  • 集成、插件、API 与扩展（7）
  • 工程化、测试与迁移（6）

这个月最重要的 7 条主线

1. APEX 26.1 不是一次小升级，而是在改开发方式

如果只看热度，5月像是在给 Oracle APEX 26.1 做集中庆典。但如果你把几篇核心文章串起来看，会发现它们讨论的不是“又多了几个功能”，而是 APEX 以后应该怎么开发、怎么协作、怎么进 Git、怎么和 AI 一起工作。

最值得看的几篇博文如下：

  • 千呼万唤始出来，Oracle APEX 26.1 正式发布！
    这篇的价值不在“宣布发布”，而在于它是这个月所有讨论的起点。

  • Oracle APEX 26.1 正式发布：APEXlang、AI Agents、DevOps 及全新应用开发方式
    这篇把 26.1 的关键词一下子抖出来了：APEXLang、AI Agents、DevOps。你会发现 Oracle 自己也不再把 APEX 只包装成“快做页面”的东西。

  • Oracle APEX 26.1新特性解析：APEXLang、.apx文件、蓝图与VS Code集成
    这是本月最值得细看的文章之一。因为它真正谈到了文件化、Git 友好、CI/CD 友好这些“团队开发级别”的变化。

  • 从低代码到AI代码：Oracle APEX 26.1解锁了什么
    这篇最值钱的地方，是它把 26.1 看成开发范式变化，而不是功能清单。

钢哥解读：APEX 26.1 之后，最值得重新评估的，不是“能不能开发企业级应用”，而是“能不能进入更正式的 AI 工程管理体系”。

2. APEXLang：这个月最像“分水岭”的关键词

如果让钢哥从5月所有话题里挑一个关键词，我会选 APEXLang。

原因很简单：很多平台都能接 AI，但不是很多平台都愿意把自己往“可文件化、可版本化、可协作”的方向重做。APEXLang 真正打动人的地方，不是名字，而是它背后那套工作方式。

值得看的博文：

  • 为 APEX 26 做准备：VS Code 中的 SQL Developer 和 Antigravity
    这是“开发环境怎么跟上变化”的视角。

  • APEXLang 来了
    适合快速建立对 APEXLang 的直观认识。

  • APEXlang的一小步
    这篇好在不空，能让你看到它怎么落地。

  • APEX 26.1 应用增强与APEXlang
    这篇说明 APEXLang 不是独立存在的噱头，而是和应用增强一起构成开发链路升级。

钢哥解读：如果您是从事企业级应用开发项目的，APEXLang 这条线比单个新功能更值得盯。因为一旦文件化和协作这件事成立，APEX 在团队里的位置都会跟着变化。

3. AI 终于不再只是“接个接口演示一下”

5月另一个非常明显的变化，是社区开始不满足于“把某个大模型 API 接进来”。讨论已经开始进入第二层：

  • 怎么做 AI Agent
  • 怎么让数据库进入 LLM 工作流
  • 怎么让自然语言查数可信
  • 怎么给 AI 代理加记忆
  • 怎么接本地模型，而不是只靠公网服务

这一组里，最值得看的有：

  • 在 Oracle APEX 中引入 AI 智能体
    这是官方口径，适合看方向。

  • 使用 UC AI 聊天插件构建 APEX AI 代理
    这篇更偏实操，告诉你代理不是 PPT 上的词。

  • 使用 LangChain 通过 SQLcl MCP 连接 Oracle AI 数据库
    这条的意义在于：数据库不是旁观者，而是被接进 Agent 工作流的核心能力。

  • SQL NL2SQL很酷，报表可信，两者可兼得！
    这篇很重要，因为它给 AI 热降了温：能问不是重点，问得准、结果可信才是重点。

  • Oracle AI数据库智能体记忆技术
    这说明讨论已经开始进入“代理怎么持续工作”这一层，而不是一次性对话演示。

钢哥解读：5月的 AEPX AI 方面最有价值的，不是热闹，而是开始从“接模型”走向“做系统”。

4. 报表这条线，也被 AI 重新定义了

很多人对 APEX 的第一印象还是报表、表单、后台系统。但5月报表相关的讨论已经开始从“会不会做报表”，转向“报表能不能被问、被筛、被解释”。

以下博文值得细品：

  • AI交互式报表 – 基于NLP的无数据泄露报表 | Oracle APEX 26.1
  • Oracle APEX 26.1 AI 交互式报告：自然语言查询功能详解
  • Oracle APEX 26.1 新功能：分面搜索排除值
  • 目录挑战：Oracle APEX 26.1 Data Reporter 对开发者的意义

钢哥解读：这几篇放一起看，会得到一个很清楚的信号：报表不再只是静态展示结果，而是变成了可对话的数据入口。 当然，这里也有边界。越是自然语言查数，越要盯住可信度、权限、解释性和误导风险，这也是为什么我觉得这组文章值得看——它们没有只讲“炫”，开始讲“稳”。

5. 安全开始补课，而且补得很实在

安全这个板块，平时最容易被忽略。因为它不够炫，也不太容易写成“新功能发布”。在企业项目里，最后真能不能上生产，往往就卡在认证、SSO、重复提交、权限边界、多标签页状态控制这些地方。

这一组里钢哥比较建议看的有：

  • APEX 自定义认证设置解析
  • 在 Oracle APEX 中使用电子邮件 OTP 构建 OTP 验证登录系统
  • 通过后台会话监控防止Oracle APEX多浏览器标签页
  • 通过自定义Claims、Scopes、Groups和JWT令牌将Okta SSO与Oracle APEX集成
  • 谁在使用你的Oracle数据（AI！）及如何确保安全

钢哥解读：这组文章说明了一件事：AI 越热，安全就越不能只停留在“以后再说”。 尤其当 APEX 开始往 AI、自然语言查询、智能体能力那边走，权限和审计就不是附属品，而是主问题。

6. 用户体验这条线，5 月其实很接地气

如果你把5月所有 APEX 新闻放一起看，会发现还有一条很容易被忽略的线：一批文章都在解决“应用能用”和“应用好用”之间的差距。比如：

  • 使用三点菜单简化APEX界面

  • Oracle APEX 表单自动保存草稿功能

  • APEX浮动操作按钮

  • Oracle APEX中Ctrl+S保存

  • 格式化Oracle APEX弹出式LOV

钢哥解读：这类文章可能不会冲上“本月最大新闻”，但对真做项目的人来说，很值钱。因为很多系统最后不是死在架构上，而是死在“用着别扭”。我会把这一组看成 APEX 生态成熟的一个信号：大家开始越来越认真地对待用户体验细节，而不是只把页面做出来就算交差。

7. 工程化、集成和测试，才是 APEX 能不能往上走的关键

最后这一组，热度不一定最高，但我反而觉得最值钱。

  • Oracle APEX + Python FastAPI：使用GitHub、Docker和Render构建云API
  • ORDS 用Podman在笔记本上搭建Oracle 26ai、ORDS和APEX 26.1本地开发栈
  • 使用 Testcontainers 进行多语言、自动化的 ORDS 测试
  • SQLCL 26.1 新特性 - DIFF 命令
  • APEX 24.1应用升级至APEX 26.1

钢哥解读：这些内容放一起，其实在回答一个更现实的问题：APEX 能不能从“一个人很快做出东西”升级为“一个团队稳定交付东西”？

这个月真正说明了什么

如果只看帖子数量，会以为 5月只是 Oracle APEX 26.1 发布后的集中转发月。但把线索串起来看，结论更有意思：APEX 的讨论重点，已经从“怎么做页面”，转向“怎么做系统”。

这个“系统”，包括：

  • 文件化和版本控制
  • AI 代理和自然语言交互
  • 数据可信与报表可解释
  • 安全、权限与审计
  • 更顺手的交互体验
  • 更正式的工程化交付

也就是说，APEX 还叫低代码，但它已经越来越像一个企业开发工作台。只是这个工作台不再只有页面能力，而是把 AI、工程化、数据交互和协作能力都往里装。这就是钢哥看完5月所有博文之后，最想先告诉你的判断。

如果你是 APEX 开发者，接下来该看什么

如果你是新手

别一上来就被 AI 和 Agent 吓住。先看 Oracle APEX 26.1 的整体变化，再理解 APEXLang 和报表能力为什么重要。

建议先看这几篇文章：

  • Oracle APEX 26.1 正式发布
  • Oracle APEX 26.1 新特性解析
  • APEXLang 来了

如果你是项目型开发者

优先看能直接改变交付方式的内容：APEXLang、报表自然语言查询、安全、自动保存、PWA、集成 API、FastAPI、测试与容器化。

如果你是团队负责人或架构负责人

你最该看的不是单个页面技巧，而是这条线：APEX 能不能纳入团队工程体系。

重点盯：

  • APEXLang / .apx / Git / VS Code
  • DevOps / SQLCL / DIFF
  • Testcontainers / ORDS 测试
  • 升级路径与本地开发栈
  • AI 引入后的安全边界

结论

5月这轮社区内容，真正值钱的地方，不是帖子多，而是方向开始收敛了。

  • 以前的 Oracle APEX 更像“做业务页面很快”。
  • 现在的 Oracle APEX 开始变成：做业务系统更完整。

而且这套“更完整”，已经不是一句宣传词了。它落在了 APEXLang、AI Agents、自然语言报表、安全控制、测试、容器、云 API 这些具体东西上。

所以如果你问我，5 月 APEX 社区最值得记住的是什么？

钢哥的答案：Oracle APEX 彻底变了，正在长成企业级应用的 AI 开发工作台。
