Oracle Application Express（APEX）19.2 正式发布了！

发表于 2019-11-02 | 更新于 2021-01-27 | 分类于 Oracle ， Oracle APEX | 阅读次数：
本文字数： 1.8k | 阅读时长 ≈ 3 分钟

Oracle Application Express（APEX）19.2正式发布了！Oracle APEX 是市场领先的企业级低代码开发平台，您可以仅使用浏览器来设计、开发和部署漂亮、响应迅速、数据驱动的桌面和移动应用程序。APEX 19.2引入了许多令人兴奋的新功能和强大的数据驱动组件，以帮助提供功能丰富且现代的用户体验。

Oracle APEX 19.2的一些重大改进包括：

分面搜索

分面搜索是 Oracle APEX 的下一个改变游戏规则的功能特性。分面搜索提供了易于使用的界面，以缩小报告的结果并快速定位特定数据。

商业网站中经常遇到分面搜索，它提供了一种直观而有效的方法，可以有效地将一组数据缩小到精确的所需子集。

新的分面搜索报表是APEX中新的声明性组件。与APEX中所有其他声明式数据驱动组件一样，您可以轻松地针对现有表、任意SQL查询、启用ORDS REST的SQL Service或几乎任何REST端点创建分面搜索报表。

使用智能的自动发现和评分算法，可以根据数据分布、相关表、列的大小、独特值、平均数据长度等自动创建推荐的分面。使用一组可靠的声明属性，还可以自定义建议的分面，并轻松添加新分面。

每个人都应该了解分面搜索特性，Oracle APEX 19.2为现代企业级应用程序提供了更加优雅和简单的分面搜索功能。

弹窗式值列表（Popup LOV）

APEX 19.2 引入了经过完全重新设计的“值列表”（LOV），新的列表控件支持 多列显示、多个返回值、按需搜索、改进的多值选择 以及全新的简化UI界面。

共享值列表（Shared LOV）

结合重新设计的弹窗值列表，APEX 19.2 中的共享值列表也通过新功能得到了显着增强。共享值列表现在支持完整的数据源集（表、SQL查询、返回SQL查询的函数、启用ORDS REST的SQL服务以及Web源模块）、声明式的列映射、多个显示列、分组、嵌入式图标等。

基于REST的交互式网格（Interactive Grid）

APEX交互式网格（Interactive Grid）是现代Web应用程序中数据感知网格组件的黄金标准。在APEX 19.2中，对启用REST的SQL服务和Web Sources的内置支持已扩展到交互网格控件（Interactive Grid）。允许您对几乎任何远程数据源进行完全读写访问！

全新的团队开发模式

APEX 19.2 完全重新设计了团队开发模式，为您的团队提供一种更加轻松简便的协作方式。无论是增强请求，新特性还是bug，都作为易于管理的问题进行跟踪、分配、标记和共享。

数据加载到现有表

APEX 19.1 引入了将数据轻松加载到Oracle数据库中的各种文件格式，包括逗号分隔的值（CSV），JSON，具有多个工作表的本机Excel（XLSX）和XML。

APEX 19.2 将此功能扩展为将数据加载到现有表中。通过相同的简单拖放界面，开发人员现在可以轻松地将其单用户桌面文件转换为基于Oracle数据库的精美Web应用程序，或将其他数据添加到现有表中。

暗模式主题样式

APEX的开发环境已经可以在“深色模式”中进行渲染，这是一种较深的配色方案，可以减轻眼睛疲劳，对于深夜开发特别有用。客户要求他们的APEX应用程序具有相同的体验。APEX 19.2 的通用主题现在包括用于“暗模式”应用程序的主题样式。

与每个APEX版本一样，许多生产力和示例应用程序也进行了增强和错误修复。此外，APEX 19.2 已修复了许多错误，从而为APEX环境提供了更高的质量和稳定性。有关APEX 19.2中许多其他令人兴奋的新功能的详细信息，请参阅“新增功能”。

附加信息

Application Express（APEX）是一种低代码快速应用程序开发平台，可以在任何Oracle数据库中运行，并且包含在每个Oracle数据库云服务中，包括Oracle自治数据库和最近发布的Oracle Cloud Free Tier。APEX与Oracle数据库相结合，提供了一个完全集成的环境，可以构建、部署、维护和监视由数据驱动的业务应用程序，这些应用程序在移动设备和台式机设备上都很不错。要了解有关 Oracle Application Express 的更多信息，请访问 apex.oracle.com。要了解有关Oracle自治数据库的更多信息，请访问 oracle.com/autonomous。
