APEX 邮件发送实践指南

发表于 2025-06-27 分类于 Oracle ， Oracle APEX 本文字数： 1.3k 阅读时长 ≈ 5 分钟

背景介绍

在设计开发企业级应用时，免不了会涉及到发邮件的场景。比如：注册验证、审批通知、系统告警等），本文将以在 Oracle Cloud 为例，介绍如何配置 Oracle Cloud Email Delivery 服务，实现通过 PL/SQL（apex_mail）原生方式发送邮件。

邮件服务配置

以 Oracle Cloud Email Delivery 为例，配置邮件服务。

启用 Email 服务 (Configuration)

登录 Oracle Cloud 控制台，切换到 Developer Services -> Email Delivery 服务，点击 Create Service 按钮，创建 Email Delivery 服务。

点击 Configuration 菜单，将 Public Endpoint、SMTP Port 记录下来。

除了上述信息外，还需要 SMTP 服务器的用户名与密码。点击右上角的用户图标，打开 profile 页面。切换到 Saved passwords 标签页，在 SMTP credentials 下，点击 Generate credentials 按钮，生成用户名与密码。

如果你的控制台用的是旧版界面，操作如下图所示：

记录下 SMTP username、SMTP password 。

钢哥提醒：这个密码只会在这里显示一次，不要忘记保存）。

添加发件人邮箱（Approved Senders）

操作路径：Developer Services -> Email Delivery -> Approved Senders, 点击 Create Approved Sender 按钮，新增发件人邮箱。如：noreply@oracleapex.cn

配置邮箱域（Email Domains）

操作路径：Developer Services -> Email Delivery -> Email Domains, 点击 Create Email Domain 按钮，新增邮箱域。如：oracleapex.cn

其中有几个比较重要的配置项，比如：DKIM、SPF，可以显著降低邮件被误判为垃圾邮件的可能性。

配置 DKIM

点击 DKIM 菜单，点击 Create DKIM 按钮，按照向导完成DKIM新增。记录下 DKIM 的 CNAME Record 和 CNAME Value。

配置 DNS 解析

登录到你的域名解析控制台（由于我的域名解析是通过阿里云域名解析，所以这里以阿里云为例）。登录阿里云控制台，切换到 云解析DNS -> 公网DNS解析 -> 权威域名解析 -> 解析设置，点击 添加记录。

填写刚才创建的 CNAME Record 和 CNAME Value 。记录类型选择 CNAME，主机记录的域名会带出来，所以只填写前面一段即可（apexcn-sg-202507._domainkey）。填写好点击 确定

同理，增加 SPF 记录，记录类型为 TXT ，记录值为：v=spf1 include:spf.oracleemaildelivery.com ~all

等待15-30分钟，刷新 Oracle Cloud 控制台，如果设置正确，状态应该显示为 active 了。

钢哥提醒：配置了 DKIM、SPF 以后，邮件被误判为垃圾邮件的概率会显著降低。

总结

APEX_MAIL 适用于大多数企业通知场景，虽然配置涉及多个组件，但一次性打通后可复用。推荐统一封装邮件发送过程，便于后期维护。
