Oracle EBS 12.2 与 APEX 增强集成正式发布！

发表于 2025-03-11 | 更新于 2025-03-12 | 分类于 Oracle ， Oracle APEX | 阅读次数：
本文字数： 1.7k | 阅读时长 ≈ 3 分钟

Oracle 正式宣布推出 Oracle EBS 12.2 全新增强功能，将 Oracle APEX 扩展嵌入到 EBS 中，这也使 Oracle APEX 正式成为 Oracle E-Business Suite (EBS) 开发扩展的首选平台。

什么是 Oracle APEX？

Oracle APEX 是 Oracle 推出的低代码应用开发平台。 APEX 将个人数据库的品质（生产力、易用性和灵活性）与企业数据库的品质（安全性、完整性、性能、可扩展性、可用性和为 Web 构建）完美结合，实现了基于浏览器的现代化UI布局、声明式的开发编程框架以及简单直观的向导，使 Oracle APEX 易于学习掌握，使您能够快速构建强大的现代化企业级应用程序。

Oracle APEX 引擎运行在 Oracle 数据库中，实现了100%基于Web浏览器进行设计、开发、测试及部署。通过 Oracle REST Data Services (ORDS) 将 http 请求进行转译，实现与数据库中的 APEX 引擎的交互。

有关 APEX 的更多信息，请参阅以下内容： Oracle APEX 产品文档。

APEX 与 Oracle E-Business Suite (EBS) 之间是什么关系？

官方推荐的 EBS-APEX 应用架构

中间层（也称为应⽤程序层）建议配置两个独立的应⽤程序服务器：⼀个⽤于 Oracle REST 数据服务，安装在 WebLogic 中，也可以安装在 Apache Tomcat 或独立部署 ORDS，但不要与现在的 EBS 共用一个 Weblogic Server。

官方推荐的 EBS-APEX 数据架构

Oracle E-Business Suite 嵌⼊模式需要在安装 EBS 的 Oracle 数据库中单独配置 Oracle APEX 对应的数据库schema（XX_APEX），并进行相应的授权。

本次发布的 EBS-APEX 集成增强都有哪些功能？

以下是新增强功能的列表，可让您开发与 EBS 环境紧密集成的扩展：

  • 使用新的 EBS 函数类型注册 APEX 页面

    • 嵌入式或全窗口是两种受支持的模式。
    • 嵌入模式允许自定义 APEX 页面出现在 OA 框架页面中，当用户在标准页面和自定义页面之间移动时提供更一致的外观和感觉。
    • 全窗口模式在单独的浏览器窗口中打开 APEX 页面。

  • 使用 EBS 配置的身份验证从 EBS 无缝导航到 APEX

    • 身份验证可以是本地身份验证，也可以是使用 Oracle Access Manager (OAM) 或 Identity Cloud Service (IDCS) 的单点登录 (SSO)。
    • 通过EBS功能安全授权来控制访问。
  • 将 APEX 工作区架构与 EBS 集成
    • APEX 架构注册为自定义 EBS 架构。
    • APEX 架构支持版本，可与 EBS 12.2 在线修补配合使用。

在哪里可以了解有关增强集成的更多信息？

现已发布一份新发布的技术论文，描述了完整的先决条件列表以及执行集成的步骤：使用 Oracle APEX 扩展 Oracle E-Business Suite 12.2 版(PDF)。

注意：增强集成的先决条件列表包括 EBS 12.2.7、Database 19c 和 APEX 24.1。请参阅技术论文，了解使用增强集成的先决条件和步骤的完整列表。
